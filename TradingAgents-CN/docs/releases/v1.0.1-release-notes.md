# TradingAgents-CN v1.0.1 发布说明

**发布日期**: 2026-04-14  
**版本类型**: Stable Release  
**版本定位**: 配置管理与数据同步体验增强版本

---

## 版本概述

`v1.0.1` 是 `v1.0.0-preview` 之后的首个稳定维护版本，重点解决了配置管理、页面切换和单股数据同步中的一批高频使用问题，目标是让系统在日常使用中更稳定、更直观、更易排查。

本版本主要聚焦以下三类改进：

1. 配置管理体验优化
2. 前端页面跳转与详情刷新修复
3. 股票实时同步兜底与日志增强
4. 人工选择性吸收上游核心能力

---

## 重点更新

### 1. 配置管理体验优化

- 新增厂家后，`模型目录管理` 页面支持按最新添加顺序置顶显示
- 新增模型后，`大模型配置` 页面支持按最新添加顺序置顶显示
- 大模型选择下拉列表支持按最新添加顺序显示
- 分析页面中的模型选择列表也统一按最新添加顺序展示
- 后端补齐大模型配置时间字段，前端排序与接口返回顺序保持一致
- 新增聚合 LLM 厂家 `AiHubMix`，支持通过统一 OpenAI 兼容接口接入多家模型
- 聚合渠道初始化接口支持自动初始化 `AiHubMix`、`OpenRouter`、`One API`、`New API` 等厂家配置

### 2. 前端页面切换修复

- 修复报告详情页在切换不同报告后仍显示旧内容的问题
- 修复股票详情页在切换不同股票后页面数据未刷新的问题
- 优化详情页状态重置逻辑，避免旧页面数据残留

### 3. 单股同步能力增强

- 增强 `/api/stock-sync/single` 的日志输出
- 同步返回结果现在会明确展示：
  - 主链路数据源
  - 回退链路数据源
  - 主错误原因
  - 回退错误原因
  - 是否已写入 `market_quotes`
- 股票详情页同步提示从简单成功提示升级为结果明细弹窗

### 4. AKShare 实时行情兜底增强

- 优化 AKShare 单股实时行情获取策略
- 新增多级降级链路：
  - `stock_bid_ask_em`
  - `stock_zh_a_spot`
  - `stock_zh_a_spot_em`
  - `stock_zh_a_hist`
- 当主接口异常时，系统会自动尝试备份接口和历史日线兜底，提升单股同步成功率

### 5. 上游项目同步能力补充

- 本版本继续采用“人工选择性吸收上游”的方式同步原项目能力，而不是整仓自动跟随
- 已进一步收敛上游 `LLM clients`、模型目录、provider 规范键和部分图层初始化路径
- 保留本项目的中文增强、Web 配置管理、MongoDB 配置体系和多市场本地化能力
- 对上游更新的处理原则是：
  - 优先吸收核心能力、底层抽象和明确缺陷修复
  - 不强行追求与上游完全一致
  - 以稳定性、收益和可维护性为优先

本次已经同步到 `v1.0.1` 的上游能力包括：

- `llm_clients` 抽象层接入主链路，统一主要 LLM 调用入口
- 共享模型目录接入 CLI 与轻量校验链路
- provider 规范键统一为 canonical key：
  - `qwen`
  - `glm`
  - `openai`
  - `google`
  - `deepseek`
  - `openrouter`
  - `ollama`
  - `qianfan`
  - `custom_openai`
- `trading_graph.py` 的主要 provider 初始化路径进一步收口到 `llm_clients`
- `fundamentals_analyst.py` 中与 qwen 相关的 fresh llm 重建逻辑适配到新路径
- 图层参数透传能力同步到当前实现
- 工厂别名兼容和风控引用修复同步到当前实现
- provider 命名兼容和依赖补齐同步到当前实现
- provider 默认 URL / 环境变量映射统一
- provider 别名归一化工具建立并接入主流程
- model catalog 校验统一到共享目录
- MongoDB 默认库名调整为 `tradingagentscn`
- 数据库支持按大版本 / 实例隔离命名
- 开发环境共享库保护加入
- MongoDB 迁移脚本、provider 规范化脚本和相关运维文档补齐

## 详细更新清单

### 配置与模型管理

- 模型目录列表支持最新厂家置顶
- 大模型配置列表支持最新模型置顶
- 大模型接口返回支持时间排序
- 模型选择对话框和分析页下拉顺序统一
- 新增 `AiHubMix` 聚合厂家支持，并纳入默认 provider 体系
- 聚合渠道初始化能力支持批量生成聚合厂家配置

### 前端交互与稳定性

- 报告详情页监听路由参数变化并自动刷新
- 股票详情页切换股票时重置旧状态并重新加载
- 同步结果弹窗支持显示详细执行结果

### 后端接口与日志

- 大模型配置增加 `created_at` / `updated_at`
- `/api/config/llm` 按最新配置优先返回
- `/api/stock-sync/single` 返回更完整的执行上下文
- 单股同步汇总日志可直接用于问题排查

### 数据同步与容错

- AKShare 单股实时行情增加备份接口
- 历史日线作为最后一级兜底
- 单股同步失败时可更快判断是网络问题、限流问题还是落库问题

### 上游能力吸收

- 延续人工选择性同步上游项目的维护策略
- 已吸收 LLM 抽象层、共享模型目录、provider 规范化、图层初始化收口和数据库迁移增强等核心能力
- 在吸收上游更新时保留中文文档体系、Web 管理能力和本地数据库配置体系
- 将上游同步过程沉淀为可追踪的维护文档和清单，便于后续版本继续演进

## 升级建议

从 `v1.0.0-preview` 升级到 `v1.0.1` 时，建议按以下步骤操作：

1. 备份当前 `.env`、数据库和自定义配置
2. 拉取最新代码
3. 重启后端与前端服务
4. 打开“配置管理”检查厂家、模型显示顺序
5. 打开任意股票详情页，执行一次“同步数据”验证同步提示和行情落库

---

## 兼容性说明

- 对现有配置数据保持兼容
- 历史大模型数据即使没有时间字段，也会按原始追加顺序倒序返回
- 不要求用户手动迁移模型配置即可使用最新排序逻辑
- 前端页面结构保持不变，主要是交互和排序行为优化

---

## 已知注意事项

- 如果 AKShare 主接口被远端断开，系统会自动尝试备份接口，但仍建议保留 Tushare 作为辅助数据源
- 如果 Tushare 账户存在频率限制，建议避免在 1 分钟内重复触发单股同步
- 历史旧数据若本身缺失实时行情记录，仍需至少成功执行一次同步才能写入 `market_quotes`
